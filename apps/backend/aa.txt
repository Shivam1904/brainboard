When the user clicks on 'Add widget' - 
We will open a form with the following fields:
Fixed for all widgets:
	Widget Type (already selected from main add widget dropdown)
	Title
	Frequency
	Importance
	Category
Fields for 'todo':
	title (has already been added)
	Todo type (habit/task)
	due date
Fields for 'alarm':
	title (has already been added)
	time of day
Fields for 'websearch':
	title (has already been added)
Fields for 'singleitemtracker':
	title (has already been added)
	value data type
	value data unit
	target value

After filling the form- the user will save - and we will call /widget/addnew api 
	this api will save in the table - DashboardWidgetDetails and specific type details table with its own fields
	To see this table data, user can go inside the widget "All schedules" and see the list of all widgets. /widget/getAllWidgetList
	User can edit these from All schedules widget using widget/updateDetails/{widget_id}

EXTERNAL APIs LIST:

At all times on load of dashboard - we will fetch the list of DailyWidgets using dashboard/getTodayWidgetList
The list of widgets will be spawned and inside each child widget, we will send the widget_id for that widget.
The widget details for today inside the widget will be fetched by /getTypeDetailsAndSummary/{widget_id} Apis for all types.

All widgets such as alarm, websearch and singleitemtracker will be able to fetch the activity details using the widget_id. 
But the habit and task list widgets will fetch their list of todos from todo/getTodayTodoList/task and todo/getTodayTodoList/habit

Inside each one's own widget, user will update status of todos, value of item tracker, etc. We will use specific type /updateActivity/{specific id} to update the activity entries.
